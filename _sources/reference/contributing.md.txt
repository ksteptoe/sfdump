# Contributing

We welcome contributions.

## Guidelines

- Follow PEP8 (enforced via pre-commit)
- Use Ruff for linting
- Write unit tests for new features
- Document new CLI commands
- Do not commit unredacted internal reports
- Ensure Sphinx documentation builds cleanly

## Pull Requests

Must include:

- Description of feature
- Tests (if applicable)
- Updated docs (for user or developer guides)
